Thesis

Why vance exists

vance is not a factory app. It is a training ground for small specialist agents that must choose tools, follow safety rules, resolve or escalate, and produce truthful reports.

Trace loop

How it works

Task -> agent attempt -> tool trace -> verifier and reward -> improved agent -> exported training data.

Post-training

Trace to data

Passing and corrected traces export as JSONL so they can become SFT or RFT examples for specialist operational agents.

Dashboard contract

What judges see

Scenario library, factory state, trace timeline, safety guardrails, reward breakdown, baseline versus improved comparison, and trace export utilities.

Sponsor stack

HUD-native eval loop

The project exposes HUD-compatible reset and step endpoints, deterministic verifier outputs, cached eval traces, and optional Fireworks/Modal paths for live model and RFT demos.

Demo video

One-minute judge flow

Open Judge Mode, run the baseline to show an unsafe shortcut, run the improved agent, inspect the trace and reward, then export the JSONL training trace.

Roadmap

After the hackathon

Expand held-out datasets, add more operational tool families, run real live-model evals, and train a small specialist policy from verified Forge-Trace style examples.
